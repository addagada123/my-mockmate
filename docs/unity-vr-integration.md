# Unity VR Integration

This repo supports a browser WebGL VR interview flow that reuses questions already generated by the normal test flow.

## Browser Flow

1. Upload a resume and generate topics/questions in the web app.
2. Open a topic test and choose a non-coding difficulty.
3. Click `Take Test in VR`.
4. The frontend calls `POST /vr-test/start` and receives a `bridge_token`.
5. The frontend loads `/vr/index.html?bridge_token=...&api_base=...`.
6. The VR page sends the payload into Unity with:

```js
unityInstance.SendMessage("MockmateVRBootstrap", "SetBridgeToken", payload);
```

## Required Scene Objects

Create these GameObjects:

1. `MockmateVRManager`
2. `MockmateVRBootstrap`

Attach these components:

### `MockmateVRManager`

- `MockmateVRApiClient`
- `MockmateVRFlowController`
- `MockmateVRAnimationBridge`
- `MockmateVRBackendTTS`
- `MockmateVRBrowserTTS`
- `MockmateVRBrowserSTT`
- `VRInterviewGlue`

### `MockmateVRBootstrap`

- `MockmateVRWebGLBootstrap`

The name `MockmateVRBootstrap` is required for browser VR startup.

## Runtime Sequence

1. `MockmateVRWebGLBootstrap.SetBridgeToken(...)` receives `bridge_token` and `api_base`.
2. It sets those values on `MockmateVRFlowController`.
3. `MockmateVRFlowController.BeginFlow()` fetches the first question from `GET /vr-bridge/next`.
4. `OnQuestionReceived` fires.
5. `VRInterviewGlue` handles question speech:
   - first `MockmateVRBackendTTS`
   - then `MockmateVRBrowserTTS` fallback
6. `MockmateVRAnimationBridge` drives talking / lip sync while the interviewer speaks.
7. `MockmateVRBrowserSTT` or your own STT path sends transcript chunks into `AppendTranscriptChunk(...)`.
8. `POST /vr-bridge/answer` submits the answer and advances to the next question.
9. `POST /vr-bridge/complete` finalizes the session.

## Backend Endpoints Used by Unity

- `GET /vr-bridge/next?bridge_token=...`
- `POST /vr-bridge/answer?bridge_token=...`
- `POST /vr-bridge/complete?bridge_token=...`
- `POST /vr-bridge/tts`
- `POST /vr-bridge/transcribe`

## Common Failure Points

- Missing or misnamed `MockmateVRBootstrap` GameObject
- `MockmateVRWebGLBootstrap` not assigned to the real `MockmateVRFlowController`
- No `AudioSource` assigned to `MockmateVRBackendTTS`
- No avatar `Animator` / jaw / face renderer wired into `MockmateVRAnimationBridge`
- STT never calling `AppendTranscriptChunk(...)`
